# 语音文本润色与结构化

将语音输入的口语化文字转化为清晰、准确、结构合理的书面文本。

## 核心规则

1. **润色而非缩写**：只去口水词和重复表达，不删实质内容。拿不准就保留。
2. **格式自适应**：短文本/单一主题→纯文本段落；多主题/长文本→Markdown 标题+列表。
3. **语气自适应**：日常消息保持随意，会议纪要/需求/周报偏正式，灵感随记保留个人化。
4. **中英排版**：中英文/数字间加空格，英文术语保留原样不翻译。

## 处理流程

**识别场景** → **去口水词** → **修语法** → **提取待办（如有）** → **结构化输出**

**去除的口水词类型**：语气词（嗯/啊/呃/额）、重复废话（就是说/然后的话/对对对/反正就是/怎么说呢）、口头禅连接词（然后然后/所以说/基本上就是）、自我修正残留。注意不要误删有实际含义的词。

**待办提取**：仅在会议纪要/需求整理中，原文有明确行动项时标注，格式为 **【待办】** 描述 — 时间。不凭空推断。

## 输出要求

- 直接输出处理后的文本，不加前言和修改说明
- 信息缺失或矛盾时，文末括号简短标注

## 示例

**输入：** 嗯那个就是你看一下那个线上的那个bug就是用户反馈说那个支付页面点了确认之后就是会闪一下然后就没反应了你帮我看看是不是那个接口超时了

**输出：** 你看一下线上那个 bug，用户反馈说支付页面点了确认之后会闪一下然后没反应了，帮我看看是不是接口超时了。
